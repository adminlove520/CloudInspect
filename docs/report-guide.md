## 风险等级

### 等级定义

| 等级 | 标识 | 退出码 | 说明 |
|---|---|---|---|
| **正常** | 🟢 | 0 | 无警告、无严重告警 |
| **警告** | 🟡 | 1 | 有警告项，需关注 |
| **严重** | 🔴 | 2 | 有严重告警，需立即处理 |

### 如何判断

```
退出码 = 0 → 正常 (全部绿色)
退出码 = 1 → 警告 (有黄色项)
退出码 = 2 → 严重 (有红色项)
```

## 各模块指标解读

### 系统信息

| 指标 | 正常范围 | 警告阈值 | 严重阈值 |
|---|---|---|---|
| CPU 使用率 | < 80% | ≥ 80% | ≥ 90% |
| 内存使用率 | < 85% | ≥ 85% | ≥ 95% |
| 负载 (1分钟) | < 核心数×2 | ≥ 核心数×2 | ≥ 核心数×4 |
| 运行时间 | - | < 1小时 | - |

### 磁盘状态

| 指标 | 正常范围 | 警告阈值 | 严重阈值 |
|---|---|---|---|
| 磁盘使用率 | < 85% | ≥ 85% | ≥ 95% |
| Inode 使用率 | < 85% | ≥ 85% | ≥ 95% |

### 网络状态

| 指标 | 正常范围 | 警告阈值 | 严重阈值 |
|---|---|---|---|
| TIME_WAIT 连接 | < 5000 | ≥ 5000 | - |
| CLOSE_WAIT 连接 | < 500 | ≥ 500 | - |
| 混杂模式 | 无 | - | 发现则严重 |

### 进程状态

| 指标 | 正常范围 | 警告阈值 | 严重阈值 |
|---|---|---|---|
| 僵尸进程 | 0 | > 0 | - |

## 告警类型解读

### 🔴 严重告警

需要立即处理的告警：

| 告警类型 | 说明 | 处理建议 |
|---|---|---|
| **LD_PRELOAD 后门** | 发现预加载后门 | 立即排查，清除后门 |
| **SSH Wrapper** | SSH 被篡改 | 重新安装 openssh-server |
| **隐藏用户** | 存在隐藏账户 | 删除可疑账户 |
| **Rootkit** | 检测到 Rootkit | 建议重装系统 |
| **反弹 Shell** | 历史中有反弹 shell | 主机可能已被控 |
| **境外成功登录** | 境外 IP 登录成功 | 确认是否合法 |
| **磁盘 95%+** | 磁盘即将耗尽 | 立即清理 |

### 🟡 警告告警

需要关注但不一定紧急：

| 告警类型 | 说明 | 处理建议 |
|---|---|---|
| **CPU 过高** | CPU > 80% | 检查高 CPU 进程 |
| **内存过高** | 内存 > 85% | 检查内存泄漏 |
| **登录失败过多** | 暴力破解迹象 | 封禁 IP，加强安全 |
| **可疑 cron** | 异常定时任务 | 检查任务内容 |
| **SElinux 未启用** | 安全加固项缺失 | 建议启用 |

## 报告示例解读

### 正常报告

```
风险等级: 🟢 正常
警告: 0 | 严重: 0
退出码: 0

所有模块检测正常，无告警项。
建议: 保持当前状态，定期巡检。
```

### 警告报告

```
风险等级: 🟡 警告
警告: 3 | 严重: 0
退出码: 1

告警项:
1. CPU 使用率 82% (超过 80% 阈值)
2. /var/log 使用率 84% (接近 85% 阈值)
3. 发现 1 个僵尸进程 (PID: 12345)

建议:
1. 检查高 CPU 进程
2. 清理 /var/log 日志
3. 处理僵尸进程
```

### 严重报告

```
风险等级: 🔴 严重
警告: 2 | 严重: 3
退出码: 2

严重告警:
1. 检测到 LD_PRELOAD 后门特征
2. 发现可疑 cron 任务 (境外下载)
3. 检测到隐藏进程

建议:
⚠️ 建议立即进入应急响应流程
1. 断开网络连接
2. 备份重要数据
3. 清除后门
4. 如有必要，重装系统
```

## 导出报告用于分析

### 导出 JSON

```bash
# 生成 JSON 报告
./inspect.sh -f json -o /tmp/report.json

# 使用 jq 分析
cat /tmp/report.json | jq '.issues'
cat /tmp/report.json | jq '.sysinfo.cpu_usage_pct'
```

### 导出 Markdown

```bash
# 生成 Markdown 报告
./inspect.sh -f md -o /tmp/report.md

# 查看告警部分
grep -A5 "告警\|警告\|严重" /tmp/report.md
```

### 批量分析

```bash
# 批量生成报告
for host in server1 server2 server3; do
  ssh $host "./inspect.sh -f json -o /tmp/report.json"
  scp $host:/tmp/report.json /tmp/$host-report.json
done

# 汇总分析
jq -s '.[] | select(.issues | length > 0)' /tmp/*-report.json
```

## 报告归档

### 按时间归档

```bash
# 创建归档目录
mkdir -p /var/log/cloudinspect/$(date +%Y-%m)

# 移动报告
mv /tmp/cloudinspect/*.html /var/log/cloudinspect/$(date +%Y-%m)/

# 压缩旧报告
find /var/log/cloudinspect -name "*.html" -mtime +30 -exec gzip {} \;
```

### Git 管理报告

```bash
# 初始化报告仓库
git init /var/log/cloudinspect-reports

# 每天提交
cd /var/log/cloudinspect-reports
git add .
git commit -m "Daily report $(date +%Y-%m-%d)"
```
